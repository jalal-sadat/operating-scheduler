What is Scheduler Design?

Scheduler Design

Scheduler Design refers to the creation and implementation of the algorithms and policies that the Operating System uses to decide which process (or thread) gets to use the CPU(Centereal Processing Unit) and for how long. The main goal is to maximize CPU utiliazatoin (keep the CPU as busy as possible) and ensure that the system is responsive, fair, and efficient for all users and process

Preemptive (Modern Way)

The OS can pause a running program to give another one a turn. Lika meeting with a timer. Everyone gets a short, fair turn to speak. This is essential for all modern computers (Windows, Mac, Linux).

Non-Preemptive (Old Way)

A program keeps the CPU until if finishes or pauses itself. Like a meeting with no breaks. One person talks until they are done.

Common Scheduling Algorithms (Kinds of Design)

First Come, First Served

First program in line is the first one run.

Shortest Job First

Picks the process with the smallest total CPU burst.

Priority Scheduling

Each program has a priority level. The highest priority runs first.

Round Robin

Every program gets a small, fixed time slice.
